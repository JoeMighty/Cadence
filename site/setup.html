Setup

Run Cadence from source

Cadence is pre-release. The one-file Windows installer arrives in Phase 7 — until then, it runs from source in about ten minutes.

What works today: the desktop shell, the local engine, and the connection between them. Music generation and voice training arrive with Phases 1 and 2 — watch the roadmap.

Install

Four steps

1

Clone the repository

Everything lives in one repository — the desktop app and the engine.

PowerShell
git clone https://github.com/JoeMighty/Cadence.git
cd Cadence
2

Start the engine

uv creates the Python environment and installs dependencies on first run. Leave this terminal open.

PowerShell
cd engine
uv sync
uv run uvicorn app.main:app --port 8000
3

Start the app

In a second terminal. The first run compiles the Rust shell, which takes a few minutes — later runs are fast.

PowerShell
cd app
npm install
npm run tauri dev
4

Verify

A Cadence window opens. The badge under the title should read Engine online — that means the shell and the engine are talking. Optionally, add a local model for lyrics later:

PowerShell · optional
ollama pull qwen3.5:9b
If something breaks

Quick checks

The badge says "Engine offline"
Make sure the engine terminal from step 2 is still running and shows no errors. Test it directly: open http://127.0.0.1:8000/ping in a browser — it should return {"status":"ok"}. If another process owns port 8000, stop it or change the port in both places.
npm run tauri dev fails to compile
Confirm Rust is installed with the MSVC toolchain: run rustup show and look for stable-x86_64-pc-windows-msvc. Visual Studio C++ Build Tools must be present — the rustup installer offers them if missing.
How do I check my GPU is visible
Run nvidia-smi in a terminal. It should list your GPU and driver version. If the command doesn't exist, install the current NVIDIA driver first. GPU generation arrives with Phase 1.
