Roadmap

Built in phases. Each one works before the next begins.

Windows first, then macOS and Linux. A phase isn't done when the code compiles — it's done when the feature runs, end to end, on a real machine.

PHASE 0 Shipped · 11 Jul 2026

Scaffolding

The skeleton: a native shell and a local engine that can talk to each other.

  • Tauri 2 desktop shell with a static Next.js frontend
  • Python FastAPI engine in its own environment
  • Rust ↔ Python communication verified over localhost
PHASE 1 Shipped · 12 Jul 2026

Engine core

The two models that make Cadence possible, working on real hardware.

  • ACE-Step 1.5 generating full tracks on CUDA
  • RVC (Applio) converting voices on a test file
  • Both wrapped behind engine endpoints with a job queue
PHASE 2 Shipped · 12 Jul 2026

Voice training

Recording and training, entirely in the app. No external tools.

  • Guided reading script with take-by-take recording
  • Live level meter — too quiet, too loud, clipping
  • Playback and keep-or-redo after every take
  • Background RVC training once 10–15 minutes are collected
PHASE 3 Shipped · 12 Jul 2026

Generate flow

The core loop: prompt in, finished track out.

  • Home screen with prompt input and language auto-detect
  • Lyrics and style structuring through Claude or a local model
  • Plain-worded status: generating, converting to your voice, done
PHASE 4 Shipped · 12 Jul 2026

Settings and keys

Provider choice and credentials, done properly.

  • API keys in the OS keychain, never plaintext
  • Text provider toggle — local Ollama or Claude
  • Local or cloud per generation, GPU status readout
PHASE 5 Shipped · 12 Jul 2026

Library

Everything you've made, in one place.

  • Past generations with playback and metadata
  • Delete and export — WAV by default, MP3 optional
PHASE 6 Shipped · 12 Jul 2026

Polish

The last ten percent that makes it feel finished.

  • Empty, error, and loading states reviewed line by line
  • Dark mode
  • Final icon and installer branding
PHASE 7 Shipped · 13 Jul 2026

Windows installer

One file, no toolchain required.

  • .msi built with the Tauri bundler, engine bundled in
  • Downloadable as a pre-release; code signing to follow
  • v0.1.1: installed app finds its models in a stable per-user folder, set up once with scripts/setup-backends.ps1
  • v0.1.2: Settings shows the version and checks GitHub for a newer release
PHASE 8 Next up

macOS and Linux

After Windows is solid, not before.

  • Apple Silicon (MPS) backend and Linux CUDA profile
  • Cross-platform GPU detection
  • CI builds for all three platforms
