Roadmap

Built in phases. Each one works before the next begins.

Windows first, then macOS and Linux. A phase isn't done when the code compiles - it's done when the feature runs, end to end, on a real machine.

DONE Shipped · Jul 2026

The full loop, on Windows

Everything from a blank machine to a finished song works today.

  • Describe a song, or paste your own lyrics - sung in a voice you trained, a generic male/female voice (the music model's own synthesized singer, not a real person), or as an instrumental
  • Songs up to 5 minutes, or an Auto length that matches the lyrics
  • Lyrics by Ollama (local, free), Claude, OpenAI, or Gemini - keys in the OS keychain
  • Tracks in Music\Cadence or any folder you choose, with optional separated stems and WAV/MP3 export
  • One .msi installer that runs its own engine, checks for updates, and walks you through setup on first launch
  • Download the installer · full changelog
NEXT Next up

macOS and Linux

After Windows is solid, not before.

  • Apple Silicon (MPS) backend and Linux CUDA profile
  • Cross-platform GPU detection and CI builds
  • A shell version of the model setup script (today's is PowerShell only)
LATER Exploring

On the bench

Being explored, not yet promised.

  • Code signing for the installer - no more SmartScreen warning
  • Silent auto-update (today Cadence only tells you a newer version exists)
  • Multiple voices in one song - tag sections with who sings them ([Verse: Anna])
  • Regenerate one section - redo just the chorus, not the whole track
  • Batch generation - queue several prompts and walk away
  • Export stems for tracks already in the Library (today stems save only at generation time)
  • Rename voice profiles
