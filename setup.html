Setup

Install Cadence

Download the Windows installer, or run from source. Either way, full music generation needs the local backends set up — the installer ships the app and engine, not the multi-gigabyte models.

Download for Windows (.msi) Release notes

Pre-release · v0.1.4 · 23 MB · unsigned, so SmartScreen warns (More info → Run anyway). The app starts and stops its own engine.

What works today: the full loop — record and train a voice, generate a song from a prompt (or paste your own lyrics) in that voice, manage the provider, API keys, and GPU status in Settings, and browse, play, and export past tracks in the Library. Dark mode and the Windows installer have shipped; cross-platform builds are next on the roadmap.

The short version

Quick start

  1. Download the .msi and install it. Windows SmartScreen warns because it's unsigned — click More info → Run anyway.
  2. Set up the models once with setup-backends.ps1 (needs git and uv) — full command in the next section.
  3. Open Cadence. Wait for the dot in the sidebar to turn green: Engine online.
  4. Type a prompt and click Generate. The first song downloads the models (~10 GB), then plays.
Step by step

Getting started

1

Download and install

Grab the installer and run it. Because it isn't code-signed yet, Windows SmartScreen shows a blue warning — click More info, then Run anyway. It installs to C:\Program Files\Cadence.

Download for Windows (.msi)
2

Set up the AI models (one time)

The installer ships the app and its engine, but not the ~10 GB music and voice models. This script installs them into %LOCALAPPDATA%\Cadence\vendor, where Cadence looks. You need git and uv on your PATH. Open PowerShell and run:

PowerShell
git clone https://github.com/JoeMighty/Cadence.git
powershell -ExecutionPolicy Bypass -File Cadence\scripts\setup-backends.ps1

Already built the backends in a source checkout? Link to them instantly instead of rebuilding: setup-backends.ps1 -LinkFrom .\engine\vendor.

3

Open Cadence and check it's live

Launch Cadence from the Start menu. It starts and stops its own engine — nothing to run in a terminal. Watch the dot at the bottom of the sidebar: it turns green “Engine online” within about 10–20 seconds.

4

Make your first song

Type a description like “a calm lo-fi piano loop”, leave the voice on Instrumental, and click Generate. The first run shows “Starting music model server” for ~20 seconds while the model loads, then downloads the model weights (~10 GB, first time only) and plays the result.

5

Optional — voice, lyrics, and better wording

Train your own voice on the Voice tab, then pick it instead of Instrumental. To write your own lyrics, open + Advanced under the prompt and use the Lyrics box (with an Insert example template) — shape it with [Verse], [Chorus], and [Bridge] tags. For sharper multilingual lyrics, add a Claude key in Settings, or install a local model:

PowerShell · optional
ollama pull qwen3.5:9b

From source

Run it in four steps

1

Clone the repository

Everything lives in one repository — the desktop app and the engine.

PowerShell
git clone https://github.com/JoeMighty/Cadence.git
cd Cadence
2

Start the engine

uv creates the Python environment and installs dependencies on first run. Leave this terminal open.

PowerShell
cd engine
uv sync
uv run uvicorn app.main:app --port 8000
3

Start the app

In a second terminal. The first run compiles the Rust shell, which takes a few minutes — later runs are fast.

PowerShell
cd app
npm install
npm run tauri dev
4

Verify

A Cadence window opens. The badge under the title should read Engine online — that means the shell and the engine are talking. Optionally, add a local model for lyrics later:

PowerShell · optional
ollama pull qwen3.5:9b
If something breaks

Quick checks

Generation fails: “the music model isn't installed”
The AI models aren't in place yet. Run the one-time setup-backends.ps1 from step 2 — it installs them into %LOCALAPPDATA%\Cadence\vendor, where Cadence looks. If you linked a source checkout with -LinkFrom and later moved or deleted it, re-run the script to relink.
The sidebar dot says “Engine offline”
The app starts its own engine, so give it 10–20 seconds after launch. If it stays offline, fully quit Cadence and reopen it. Still stuck? Open Task Manager → Details and end any lingering cadence-engine.exe, then relaunch. (Running from source instead? Make sure your uv run uvicorn terminal is up and nothing else owns port 8000.)
Windows blocked the installer (SmartScreen)
The installer isn't code-signed yet, so Windows shows a blue “Windows protected your PC” screen. Click More info, then Run anyway. Code signing is on the list.
How do I update to a newer version?
Open Settings → About and click Check for updates. If a newer release exists it shows a Download button; install it over the top — your library and trained voices are kept separately and carry over.
How do I check my GPU is visible?
Run nvidia-smi in a terminal. It should list your GPU and driver version. If the command doesn't exist, install the current NVIDIA driver first. Without a CUDA GPU, generation falls back to CPU and takes minutes instead of seconds.
Building from source: npm run tauri dev won't compile
Confirm Rust has the MSVC toolchain: run rustup show and look for stable-x86_64-pc-windows-msvc. Visual Studio C++ Build Tools must be present — the rustup installer offers them if they're missing.
